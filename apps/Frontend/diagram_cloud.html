MTSAi Cloud Deployment Model

The MTSAi platform is designed for cloud-native deployment with India-resident data, integrating government digital infrastructure at every layer. This diagram illustrates the logical deployment architecture.

Layer 1 — Government Access
Policy Control & Governance Portal
Policy Rule Engine
Audit Dashboard
Evidence Vault
Appeal Workflow
Grievance Portal

Government officials access policy controls, set incentive parameters, review audit logs, and manage commuter appeals entirely through dedicated portals with role-based access controls.

↕
Layer 2 — Core Platform Services
Demand Management Engine
Incentive Calculator
Occupancy Verifier
Corridor Scheduler
Payout Manager
Peak Window Controller

The core engine processes vehicle data, verifies occupancy signals, applies policy rules, calculates incentive amounts, and queues payouts — all within auditable, timestamped transactions.

↕
Layer 3 — Data & Audit Layer
Immutable Audit Trail & Data Residency
Audit Hash Chain
Event Ledger
DPDP Retention Engine
Data Residency (IN)

All transactions are written to an append-only audit log with cryptographic hashing. Data is retained on India-resident infrastructure in compliance with DPDP Act provisions.

↕
Layer 4 — Government Infrastructure Integration
India Digital Public Infrastructure Connectors
FASTag (NETC)
UPI Disbursement
Aadhaar eKYC
ICCC API
ANPR / LPR

MTSAi integrates with India's existing public digital infrastructure — FASTag for vehicle identification, UPI for incentive disbursement, Aadhaar eKYC for commuter verification, and ICCCs for real-time traffic data.

↕
Layer 5 — Security & Compliance
CERT-In & ISO 27001 Baseline
CERT-In Guidelines
Penetration Testing
Incident Response
Access Controls (RBAC)
Encryption at Rest

Security posture designed to CERT-In and ISO 27001 requirements, with encryption at rest and in transit, role-based access controls, and documented incident response procedures.

↕
Layer 6 — Cloud Infrastructure
India-Resident Cloud Deployment
MeitY Cloud Framework
India Region (IN-WEST-1)
Multi-AZ Failover
Gov-Auditable Hosting

Deployed on MeitY-empanelled cloud infrastructure within India, with multi-availability-zone resilience and government-auditable hosting configurations.

Data Residency
India-Resident Storage

All personally identifiable data and transaction records stored within India. Compliant with DPDP Act data localisation provisions as designed.

Integration Model
API-First Architecture

All integrations with FASTag, UPI, ICCC, and ANPR systems use documented REST APIs with versioned contracts and government-controlled credentials.

Deployment Model
City-Specific Tenancy

Each city deployment operates as a separate logical tenant. Policy rules, data, and audit logs are fully isolated per procuring authority.

Architecture Notice: This diagram represents the intended deployment architecture as a design specification. Final hosting model, integration scope, and security certification are confirmed through the procuring entity's RFP and technical due diligence process. No live deployments are currently operational.
